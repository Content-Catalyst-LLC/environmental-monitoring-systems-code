# Platform Evidence Quality Model Card

## Model purpose

This placeholder model card documents a platform evidence-quality scoring workflow.

## Intended use

- evaluate metadata completeness
- identify semantic integration risks
- flag lineage gaps
- flag hidden uncertainty
- assess decision-support fit
- triage high-stakes evidence assets
- support platform governance reviews

## Not intended for

- replacing scientific or domain review
- certifying platform outputs for public use without governance approval
- scoring environmental harm directly
- treating data availability as decision readiness
- hiding weak evidence behind composite scores

## Inputs

- metadata completeness
- semantic compatibility
- lineage completeness
- uncertainty visibility
- decision fit
- high-stakes status

## Known limitations

- simplified scoring
- domain-specific review required
- semantic compatibility requires expert validation
- uncertainty visibility does not guarantee uncertainty quality
- decision fit depends on user role and institutional context
