# Deployment Readiness Gate

Before an environmental data platform or decision support system is used for operations, public communication, planning, regulatory review, funding decisions, modeling, or accountability, it should pass a readiness gate.

## Readiness checks

1. Platform purpose and decision context are clearly defined.
2. Source inventory includes owner, cadence, domain, method, and access status.
3. Metadata schema includes source, method, unit, temporal scope, spatial frame, quality, provenance, and uncertainty.
4. Semantic crosswalks and unit registries validate integration assumptions.
5. Platform outputs are traceable to source records and transformations.
6. Modeled, estimated, interpolated, forecast, scenario, and administrative outputs are clearly labeled.
7. Model and scenario registries document assumptions, versions, validation, limitations, and intended use.
8. Decision-support views are aligned with real user tasks.
9. Governance policy defines stewardship, access, review, versioning, and public accountability.
10. Public evidence packages exist for material decisions or public-facing claims.
