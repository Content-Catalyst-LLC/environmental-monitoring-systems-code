# Sensor Network Evidence Framework

Environmental sensor networks are infrastructures of selective environmental visibility.

## Core layers

| Layer | Question | Evidence artifact |
|---|---|---|
| Objective | What environmental process, spatial scale, temporal scale, decision use, and evidence standard are being supported? | Network objective manifest |
| Topology | Where are nodes placed, what gradients are captured, and what blind spots remain? | Topology plan and coverage map |
| Node | Which sensors, sites, observed properties, units, calibration states, and owners define the network? | Node and site registry |
| Telemetry | How do observations move from field nodes to platforms, and how complete and timely is delivery? | Telemetry and latency manifest |
| Metadata | Can records be interpreted across nodes, vendors, years, and institutions? | Observation schema and metadata registry |
| QA/QC | How are range, spike, flatline, drift, missingness, calibration, and cross-node consistency handled? | Network QA/QC policy |
| Platform | How are data stored, harmonized, accessed, and reused? | API, data catalog, and lineage log |
| Alert | What thresholds, severity levels, escalation pathways, and response owners make data actionable? | Alert-rule registry |
| Governance | Who maintains, audits, revises, explains, and publicly accounts for the observing system? | Governance log and public evidence package |
