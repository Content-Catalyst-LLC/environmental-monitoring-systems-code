# IoT Architecture Readiness Model Card

## Purpose

Evaluate environmental IoT architecture readiness across device availability, delivery completeness, semantic completeness, quality-control readiness, edge traceability, decision fit, and latency penalty.

## Intended use

- triage architecture components
- identify high-stakes QC gaps
- review semantic completeness
- review latency and connectivity risk
- flag edge opacity
- support governance review

## Not intended for

- replacing engineering review
- replacing cybersecurity assessment
- certifying public alerts without governance review
- treating connectivity as evidence quality
