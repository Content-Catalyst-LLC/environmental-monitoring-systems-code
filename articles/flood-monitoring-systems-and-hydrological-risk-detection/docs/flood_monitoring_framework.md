# Flood Monitoring Evidence Framework

| Layer | Question | Evidence artifact |
|---|---|---|
| Objective | Which flood type, geography, protected population, lead-time goal, and decision use are being supported? | Flood-risk objective manifest |
| Observation | Which gauges, rainfall products, reservoir records, soil-wetness data, snowmelt data, and local observations support the warning chain? | Gauge and sensor registry |
| Drivers | Which hydrometeorological and infrastructure drivers shape flood generation? | Hydrometeorological driver matrix |
| Telemetry | Are observations timely, complete, quality-flagged, and not stale? | Telemetry status records |
| Forecast | Which models, horizons, guidance products, and uncertainty statements are used? | Forecast model card |
| Threshold | Which action stages, flood stages, impact thresholds, and response triggers apply? | Threshold and impact registry |
| Mapping | How are stages or forecasts translated into inundation extent and exposed assets? | Inundation-map linkage |
| Warning | Who receives alerts, through which channels, and with what action guidance? | Warning protocol and response log |
| Governance | How are coverage gaps, equity, false alarms, missed events, threshold revisions, and post-event learning handled? | Governance log |
