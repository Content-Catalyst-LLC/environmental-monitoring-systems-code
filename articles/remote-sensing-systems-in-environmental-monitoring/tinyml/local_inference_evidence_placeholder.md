# TinyML Local Inference Evidence Placeholder

This directory is reserved for a future TinyML example that produces local evidence to complement remote-sensing products.

Possible use cases:

- local flood confirmation
- vegetation stress classification
- smoke or air-quality anomaly detection
- restoration plot condition
- field camera quality flagging
- local habitat condition inference

The TinyML workflow should preserve:

- station ID
- linked remote-sensing product ID
- model ID
- model version
- inference confidence
- observed versus inferred distinction
- uncertainty label
- timestamp
- evidence-retention policy
