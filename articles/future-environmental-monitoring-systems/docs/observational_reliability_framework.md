# Observational Reliability Framework

Environmental intelligence should be treated as accountable observation.

## Reliability dimensions

| Dimension | Question | Evidence artifact |
|---|---|---|
| Measurement validity | Are observations valid for the intended use? | Calibration records, QA/QC flags |
| Coverage | Are relevant places, times, and populations represented? | Coverage audit |
| Freshness | Are observations timely enough for the decision? | Latency report |
| Interoperability | Can systems exchange and interpret data correctly? | API tests, schema validation |
| Provenance | Can outputs be traced back to sources and transformations? | Lineage graph, workflow manifest |
| Model trust | Has the AI model been validated in the intended domain? | Model card, validation report |
| Governance | Are access, review, accountability, and equity addressed? | Governance policy |
| Decision usefulness | Can users act responsibly on the output? | Decision guide, threshold rationale |
