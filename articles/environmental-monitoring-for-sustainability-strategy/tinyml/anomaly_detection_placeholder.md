# TinyML Anomaly Detection Placeholder

This directory is reserved for a future on-device anomaly detection example for sustainability monitoring.

Possible use cases:

- water-quality anomaly detection
- air-quality threshold alert
- soil-moisture stress signal
- waste-stream anomaly detection
- local sensor drift detection

The TinyML workflow should preserve:

- observed versus inferred distinction
- indicator ID
- threshold version
- confidence score
- firmware version
- evidence retention policy
