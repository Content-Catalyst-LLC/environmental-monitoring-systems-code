# Biodiversity Monitoring Evidence Framework

Biodiversity monitoring systems are ecological knowledge infrastructures that transform partial observations into evidence for conservation, restoration, policy, and accountability.

## Core layers

| Layer | Question | Evidence artifact |
|---|---|---|
| Objective | Which taxa, habitats, geography, temporal window, decision use, and reporting framework are being monitored? | Monitoring objective manifest |
| Dimension | Which biodiversity dimensions are represented: genes, populations, species, distributions, communities, habitat, ecosystem function, or pressures? | Biodiversity dimension matrix |
| Method | Which methods are used, and what does each method detect or miss? | Observation method registry |
| Observation | Which records, effort, site metadata, and quality flags support inference? | Species observation records |
| Detectability | How are non-detection, effort, season, sensor placement, and method bias handled? | Detectability audit |
| Indicator | Which EBVs, GBF indicators, or conservation metrics are derived from observations? | Indicator registry |
| Habitat context | How are species observations connected to habitat extent, structure, fragmentation, and ecosystem condition? | Habitat-context linkage file |
| Inclusion | How are community-based, Indigenous, local, and citizen-science observations governed? | Knowledge governance protocol |
| Reporting | Which policy or stewardship claims can the evidence support? | Reporting readiness and public evidence package |
| Governance | Who owns revisions, caveats, validation, access, and accountability decisions? | Governance log |
