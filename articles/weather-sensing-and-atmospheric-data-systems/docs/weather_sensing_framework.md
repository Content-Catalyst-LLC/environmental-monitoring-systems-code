# Weather Sensing Evidence Framework

| Layer | Question | Evidence artifact |
|---|---|---|
| Objective | Which forecast, nowcast, warning, aviation, marine, hydrological, or decision-support use case is being supported? | Weather sensing objective manifest |
| Platform | Which observing systems produce the relevant evidence? | Observation platform registry |
| Observation | What was observed, where, when, by which platform, with what unit and quality status? | Atmospheric observation records |
| Telemetry | Did expected observations arrive within the operational latency budget? | Telemetry and latency log |
| Quality control | Were range, temporal, internal consistency, spatial, and station-health checks applied? | Quality-control policy and QC flag table |
| Interoperability | Can observations be exchanged, decoded, standardized, and reused across systems? | Schema, data dictionary, exchange-format notes |
| Assimilation | Can valid observations enter model, nowcast, or analysis workflows? | Assimilation readiness records |
| Coverage | Which spatial, vertical, marine, radar, satellite, or institutional gaps remain? | Coverage and blind-spot audit |
| Forecast use | Which observations supported forecast, nowcast, warning, or operational decisions? | Forecast-use log |
| Governance | Who maintains, shares, audits, and revises the observation system? | Governance and data-sharing log |
