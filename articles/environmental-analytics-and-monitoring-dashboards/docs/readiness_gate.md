# Deployment Readiness Gate

Before an environmental dashboard is used for operations, regulatory oversight, public communication, planning, emergency response, or accountability, it should pass a readiness gate.

## Readiness checks

1. Dashboard purpose and user roles are clearly defined.
2. Data sources are documented, fresh, validated, and quality-flagged.
3. Indicators include units, baselines, thresholds, methods, uncertainty, and owners.
4. Map layers include timestamp, resolution, CRS, source, method, and confidence.
5. Alerts are actionable and connected to escalation rules.
6. Modeled, estimated, interpolated, and low-confidence values are labeled.
7. Users can drill down from summaries to source evidence.
8. Visual encodings are accessible and support valid comparison.
9. Dashboard changes are documented and reviewed.
10. User testing confirms that target users can interpret and act on dashboard outputs.
