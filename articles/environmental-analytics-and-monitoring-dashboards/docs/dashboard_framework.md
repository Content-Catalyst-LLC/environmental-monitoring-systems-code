# Environmental Dashboard Framework

Environmental dashboards are infrastructures of attention. They structure what users see, compare, trust, escalate, and act upon.

## Core layers

| Layer | Question | Evidence artifact |
|---|---|---|
| Source | Where do observations, models, records, and geospatial layers originate? | Source inventory |
| Validation | Are data fresh, complete, unit-consistent, and quality-flagged? | QA/QC report |
| Transformation | How do source data become displayed indicators and layers? | Transformation manifest |
| Analytics | What thresholds, anomalies, trends, and comparisons are computed? | Indicator and alert registry |
| Interface | How are maps, charts, alerts, filters, and cards presented? | Visualization specification |
| Interpretation | Can users understand context, uncertainty, provenance, and action? | Method notes and provenance links |
| Governance | Who owns indicators, thresholds, views, and dashboard changes? | Governance policy and change log |
| Learning | How do user feedback and event review improve the dashboard? | Review backlog and after-action record |
