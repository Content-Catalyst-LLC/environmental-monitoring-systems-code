# Dashboard Signal Quality Model Card

## Model purpose

This placeholder model card documents a dashboard signal-quality and alert-review workflow.

## Intended use

- evaluate dashboard indicator freshness
- assess provenance completeness
- identify hidden uncertainty
- flag alert burden
- review user-role fit
- support dashboard governance
- support evidence reconstruction

## Not intended for

- automatic public warning without human review
- hiding weak evidence behind dashboard scores
- replacing domain expert judgment
- ranking environmental risk without context
- treating visual salience as evidentiary strength

## Inputs

- freshness status
- provenance status
- modeled or estimated status
- uncertainty-label status
- user-role fit
- alert count
- actionable alert count

## Known limitations

- simplified scoring
- context-specific role fit required
- alert actionability depends on institutional process
- provenance completeness does not guarantee source validity
- uncertainty visibility must be interpreted by domain users
