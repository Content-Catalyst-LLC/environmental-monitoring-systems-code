# TinyML Anomaly Detection Placeholder

This directory is reserved for a future on-device anomaly detection example for early warning networks.

Possible use cases:

- rapid water-level change
- heat threshold warning
- smoke or air-quality anomaly
- tsunami signal prefilter
- local sensor failure detection

The TinyML workflow should preserve:

- observed versus inferred distinction
- confidence score
- threshold version
- firmware version
- event evidence retention policy
