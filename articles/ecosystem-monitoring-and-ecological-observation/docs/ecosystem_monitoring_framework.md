# Ecosystem Monitoring Evidence Framework

Ecosystem monitoring is ecological knowledge infrastructure.

## Core layers

| Layer | Question | Evidence artifact |
|---|---|---|
| Objective | What ecosystem type, geography, baseline, temporal window, decision use, and evidence standard are being monitored? | Monitoring objective manifest |
| Dimension | Which dimensions are represented: extent, condition, structure, function, integrity, resilience, disturbance, and recovery? | Ecosystem dimension matrix |
| Observation | Which field, sensor, remote-sensing, biological, hydrological, and contextual observations support the assessment? | Ecological observation registry |
| Indicator | Which indicators translate observations into ecological meaning, and what are their proxy limits? | Ecological indicator registry |
| Baseline | What reference condition, benchmark, control, or historical record anchors interpretation? | Baseline and reference statement |
| Disturbance | Which disturbances, restoration actions, management changes, and recovery pathways shape interpretation? | Disturbance and recovery log |
| Assessment | How are indicators synthesized into condition, integrity, resilience, recovery, or risk judgments? | Assessment score table and model card |
| Validation | How are observations, indicators, and composite claims tested? | QA/QC and validation report |
| Representativeness | What ecosystem types, gradients, under-monitored places, and risk zones remain weakly observed? | Representativeness audit |
| Governance | Who owns revisions, public caveats, stewardship decisions, and disputed claims? | Governance log |
